> 讨论稿：这份文档保留 Hermes 早期规划。正式 V7 阶段拆分和安全边界以 `docs/V7_PLAN.md` 为准。

## 阶段目标

从CheckpointAI V6之后，下一步是V7：Autonomous Loop。

核心目标：让系统能自动发现问题、分析原因、生成建议、执行验证。

---

## V7.1 Alpha：第一阶段

### 目标

```
- 3个Agent能跑起来
- 对接一个外置系统（简单优先）
- 验证闭环能工作
```

### 3个Agent（合并后）

```
1. Investor（调查者）
   = 主动探索 + 监控异常
   - 发现数据异常
   - 分析可能原因
   - 输出：异常报告

2. Proposer（建议者）
   - 分析原因
   - 生成proposal
   - 评估风险
   - 输出：proposal

3. Refiner（优化者）
   - 执行proposal
   - 验证效果
   - 更新知识库
   - 输出：验证报告
```

### Safety Agent（安全员）

```
独立于闭环之外：
- 监控执行
- 发现异常触发回滚
- 记录失败模式
```

### 外置系统对接

```
第一阶段选简单的：
- 已有Adapter的系统
- API接口简单
- 返回结果明确

需要统一接口：
- 外置Agent执行结果 → 统一格式 → UI可查看
```

### 改动策略

```
不冲突的改动 → 一起执行
冲突的改动 → 人来决定

一次只做一个改动
改动尽量小
```

## 自动化程度渐进

```
第一阶段：全部人审批
- 系统只是记录和验证
- 验证闭环能跑

第二阶段：低风险自动执行
- 高风险人审批
- 验证置信度模型

第三阶段：根据置信度自动执行
- 极高置信度才自动
- 持续学习改进
```

---

## 执行顺序

```
Step 1：实现3个Agent
- Investor
- Proposer
- Refiner

Step 2：实现Safety Agent
- 回滚机制
- 异常监控

Step 3：对接第一个外置系统
- Adapter契约
- 统一结果接口

Step 4：验证闭环
- 跑数据
- 验证效果
- 调整参数

Step 5：扩展到更多外置系统
```

---

## 怎么知道系统在变好

```
需要指标：
- 优化次数/天
- 改动成功率
- 平均效果提升
- 回滚次数

如果指标没改善：
- 说明系统在乱改
- 需要调整约束
```

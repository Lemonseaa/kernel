## 阶段1：模板系统

### 任务描述

实现模板注册表和模板应用器，支持BusinessLine基于模板创建。

### 文件结构

```
opc_os/templates/
__init__.py
models.py        # AgentTemplate, WorkflowTemplate, BusinessLineTemplate
registry.py     # TemplateRegistry
applier.py      # TemplateApplier
defaults.py     # 内置模板：blank, content, website
```

### 测试要求

- 模板注册和查询
- 内置模板加载
- 基于模板创建BusinessLine

### 验收标准

- 能列出所有模板
- 能基于content模板创建业务线
- 模板包含预设的Agent、Evaluation规则、Policy

---

## 阶段2：Policy继承/覆盖

### 任务描述

PolicyEngine支持Global Policy和BusinessLine Policy两层，BusinessLine Policy优先于Global Policy。

### 文件结构

```
opc_os/control/
defaults.py     # 内置Policy：high_risk_require_approval, publish_require_approval
```

修改已有文件：
- opc_os/control/policy.py — 加scope、add_bl_policy、BL优先逻辑

### 测试要求

- Global Policy对所有业务线生效
- BusinessLine Policy能覆盖Global Policy
- 未配置BL Policy时回退到Global Policy

### 验收标准

- 高风险工具默认需要审批（Global）
- BL可以添加自己的Policy覆盖Global
- Policy按优先级：BL Policy > Global Policy > ALLOW

---

## 阶段3：成本预算 + HumanGate联动

### 任务描述

CostTracker超预算时发布事件，HumanApprovalGate订阅该事件并创建审批任务。

### 修改文件

- opc_os/observability/cost_tracker.py — 已有功能，可能需要增强
- opc_os/control/human_gate.py — 订阅cost.budget_exceeded事件

### 测试要求

- 设置预算后，超预算能触发审批
- 审批队列能看到成本超预算任务

### 验收标准

- set_cost_budget设置每日预算
- 消耗超预算时，HumanGate收到cost.budget_exceeded事件
- 审批队列包含成本审批任务

---

## 阶段4：Context持久化（可选）

### 任务描述

BusinessLine支持跨Run持久化Context，Run结束后数据不丢失。

### 文件结构

```
opc_os/context/
persistent.py   # PersistentContext
```

### 测试要求

- 数据能跨Run保留
- 不同BusinessLine的Context互不影响

### 验收标准

- Run结束后Context数据仍然存在
- 新Run能读取上一Run的数据
